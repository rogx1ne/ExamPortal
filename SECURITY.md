# Security Policy

## Reporting a vulnerability

If you believe you have found a security issue, please do **not** open a public issue.

Preferred: use your Git hosting provider's private vulnerability reporting / Security Advisories feature.

If private reporting is not available, contact the project maintainer privately and include:

- A clear description of the issue and impact
- Steps to reproduce (proof-of-concept if possible)
- Affected versions / commit SHA (if known)
